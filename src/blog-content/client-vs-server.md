## Client vs Server

Every app you use today — from Instagram to banking apps — works because of one simple relationship:

Client 🤝 Server

Understanding this is the foundation of how the internet works.

A **Client** is anything that requests something.

This could be:
- Your browser
- A mobile app
- A frontend application

Whenever you click a button, load a page, or submit a form — the client sends a request.

On the other side, the **Server** is responsible for responding.

It:
- Processes logic
- Fetches data
- Talks to databases
- Sends back results

Think of it like this:

Client = The one who asks  
Server = The one who answers  

For example:

When you log into a website —

Your frontend sends:


Email + Password


The server checks:
- Is this user valid?
- Does the password match?
- What data should be returned?

Then it responds with:
- Success / Failure
- User data
- Authentication token

And your UI updates accordingly.

This constant request–response cycle is what powers everything on the internet.

Without clients asking…  
Servers would do nothing.

Without servers responding…  
Clients would show nothing.

Every modern application is simply a conversation between these two.
